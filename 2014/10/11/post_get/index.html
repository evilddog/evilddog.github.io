浅谈HTTP方法中的POST和GET请求

最近协会给小鲜肉布置渗透的作业有提到HTTP请求中的POST和GET请求,所以在这里复习一下

众所周知，在客户机和服务器之间进行请求响应是最常用的方法是GET和POST

  • GET - 从指定的资源请求数据。
  • POST - 向指定的资源提交要被处理的数据

GET方法 查询的key&value是在GET请求的URL中发送的：

/test/demo_form.asp?name1=value1&name2=value2

有关 GET 请求的其他一些特征：

  • GET 请求可被缓存

  • GET 请求保留在浏览器历史记录中

  • GET 请求可被收藏为书签

  • GET 请求不应在处理敏感数据时使用

  • GET 请求有长度限制

  • GET 请求只应当用于取回数据

POST方法查询的Key&value是在POST请求的HTTP消息主体中发送的

POST test demo_form.asp 
HTTP/1.1 
Host:evilddog.github.io
name1=value1&name2=value2

有关 POST 请求的其他一些特征：

  • POST 请求不会被缓存

  • POST 请求不会保留在浏览器历史记录中

  • POST 不能被收藏为书签

  • POST 请求对数据长度没有要求
